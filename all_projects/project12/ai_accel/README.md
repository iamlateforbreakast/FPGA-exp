# Tiny GRU Story Accelerator
### Tang Nano 20K FPGA + Raspberry Pi + OCaml/HardCaml

A complete DIY AI accelerator that generates short stories by running a 
character-level GRU neural network on an FPGA, controlled by a Raspberry Pi.

## Phase 1 вҖ” Train the Model (on your PC)

### Requirements
```bash
pip install torch numpy
```

### Run
```bash
cd train/
python train.py
```

This trains for ~300 epochs on Aesop's Fables, then exports:
- `weights/*.bin`  вҖ” INT8 quantized weight matrices
- `weights/vocab.json` вҖ” character vocabulary
- `weights/meta.json`  вҖ” shapes and quantisation scales

**Expected output:**
```
Vocab size: 62
Epoch  50  loss=2.8431
Epoch 100  loss=2.1205
Epoch 150  loss=1.8034
...
Sample (float32):
A fox and the lion was a mouse...
Exporting INT8 weights:
  embed                  shape=(62, 128)  scale=0.012543
  gru_weight_ih          shape=(384, 128) scale=0.008921
  ...
```

**GRU model parameters:**
| Parameter | Value |
|-----------|-------|
| Vocab size | ~62 chars |
| Hidden size | 128 |
| GRU layers | 1 |
| Total weights | ~200 KB (float32) вҶ’ ~50 KB (INT8) |
| BRAM needed | ~50 KB вҶ’ fits in Tang Nano 20K's 828 KB BRAM |

### Key modules in gru_core.ml

| Module | Description |
|--------|-------------|
| `Mac_unit` | Serialised INT8Г—INT8вҶ’INT32 multiply-accumulate |
| `sigmoid_lut` | 256-entry piecewise-linear sigmoid ROM |
| `tanh_lut` | 256-entry piecewise-linear tanh ROM |
| `create` (top) | GRU cell state machine + SPI slave |

### State machine
```
IDLE вҶ’ LOAD_X вҶ’ GATE_R вҶ’ GATE_Z вҶ’ GATE_N вҶ’ UPDATE_H вҶ’ OUTPUT вҶ’ IDLE
                  вҶ‘_________128 MAC cycles each__________вҶ‘
```

### Simulate
```bash
cd fpga/lib/
dune exec ./sim.exe
# Prints: Token out: 23  (or similar)
```

### Synthesise (Gowin IDE)
1. Open Gowin IDE, target: GW2AR-18 (Tang Nano 20K)
2. Add `gru_core.ml` вҖ” first run HardCaml to emit Verilog:
```bash
dune exec ./sim.exe -- --rtl-only -o gru_core.v
```
3. Add generated `gru_core.v` to Gowin project
4. Add constraint file for SPI pins (see below)
5. Synthesise вҶ’ Place & Route вҶ’ Generate bitstream
6. Flash via openFPGALoader:
```bash
openFPGALoader -b tangnano20k gru_core.fs
```

### Pin constraints (Tang Nano 20K)
```
// SPI to Raspberry Pi
IO_LOC "spi_sclk"  48;  // GPIO 11 (SPI CLK)
IO_LOC "spi_mosi"  47;  // GPIO 10 (SPI MOSI)
IO_LOC "spi_miso"  46;  // GPIO  9 (SPI MISO)
IO_LOC "spi_cs_n"  45;  // GPIO  8 (SPI CE0)
IO_LOC "reset"     15;  // GPIO 25 (reset line)
```

## SPI Protocol

| Direction | Byte | Meaning |
|-----------|------|---------|
| Pi вҶ’ FPGA | `0x01` | CMD_START вҖ” begin inference step |
| Pi вҶ’ FPGA | `0xXX` | Character index (vocab-encoded) |
| FPGA вҶ’ Pi | `0xFF` | Busy вҖ” FPGA still computing |
| FPGA вҶ’ Pi | `0xXX` | Token index вҖ” inference complete |
| Pi вҶ’ FPGA | `0x02` | CMD_NEXT вҖ” advance one token |
| Pi вҶ’ FPGA | `0x00` | CMD_STOP вҖ” reset FPGA state |

## Resource Usage Estimates (Tang Nano 20K)

| Resource | Used | Available | % |
|----------|------|-----------|---|
| LUT4 | ~8,500 | 20,736 | 41% |
| Flip-Flops | ~3,200 | 15,552 | 21% |
| BRAM (9K) | 46 KB | 828 KB | 6% |
| DSP (18Г—18) | 12 | 28 | 43% |

The design is comfortably within budget. You could scale to hidden_size=256 
if desired.

## Learning Checkpoints

After each phase you'll understand:

1. **Phase 1** вҖ” How neural language models work, what quantisation does, why INT8 is ~4Г— smaller than FP32
2. **Phase 2 (MAC unit)** вҖ” The core operation of every AI accelerator: multiply-accumulate
3. **Phase 2 (LUTs)** вҖ” How non-linear activations are approximated in hardware
4. **Phase 2 (state machine)** вҖ” How to sequence computation that can't all happen in one clock cycle
5. **Phase 3** вҖ” How the CPU/host side of an AI accelerator works (this is what drivers do for GPUs)

## Extending the Project

- **Bigger model**: Increase `HIDDEN_SIZE` to 256 вҖ” still fits in BRAM
- **Temperature control**: Add a register the Pi can write to control randomness
- **Beam search**: Run 2вҖ“4 GRU cells in parallel on the FPGA for better quality
- **UART fallback**: Replace SPI with UART for simpler wiring during development
- **Waveform debug**: Use `hardcaml_waveterm` to view signals during simulation
